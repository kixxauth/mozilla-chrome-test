Documentation.

Chrome Tester provides a way for you to easily install test files into the chrome of the Chrome Tester extension. Currently installed files are listed above. Choose one by clicking on it, and it will open in a new tab.

This tool greatly simplifies testing parts of your program that require chrome privileges within the Mozilla platform including access to the Components object.

For the sake of simplicity, files are made available in two forms - "chrome://" and "resource://". You designate which type the file will be by choosing either the "install chrome://" or "install resource://" button when installing. Typically content files like .xul, .html, and .css files should be loaded as "chrome://" and JavaScript modules and other resources should be loaded as "resources://".

Also, the Chrome List extension makes a very nice companion tool.
